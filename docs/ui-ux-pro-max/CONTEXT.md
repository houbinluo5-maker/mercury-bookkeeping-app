# UI/UX Pro Max Context

## App Context
Next.js, TypeScript, Tailwind bookkeeping app for a US LLC ecommerce workflow. Existing functionality includes ADMIN_PASSWORD login, Supabase persistence, receipt uploads, transactions, Mercury CSV import, reconciliation, audit trail, monthly closing, reports, tax package, and English/Simplified Chinese switching.

## Priority Workflows
- Founder/operator checks finance health and close readiness.
- Bookkeeper clears receipts, review flags, duplicates, and reconciliation blockers.
- CPA receives export-ready tax package and audit context.
- Admin reviews Supabase status and local backup tools safely.

## Current UI Gap
The current app is serviceable but not commercial enough. It needs sharper hierarchy, stronger product framing, better panels, more deliberate data grouping, and a more premium finance SaaS visual language.

## Constraints
- Preserve existing business logic and protected workflows.
- Use existing stack and Tailwind.
- Prefer reusable components and global classes.
- Keep changes incremental and mergeable.
